# Ideation: oh-my-stock

## Core Idea
- 한국 개인투자자가 KR/US 종목 급등락을 봤을 때, 여러 앱을 오가며 원인을 찾는 시간을 1분 이내로 줄이는 "근거 기반 이벤트 설명" 서비스다.
- 핵심 산출물은 가격 알림이 아니라 `원인 카드`다: 후보 1~3개, 신뢰도, 근거 URL, 업데이트 시각, 사용자 포트폴리오 영향.
- 왜 지금:
  - 한국 개인투자자 모수가 크고(2025년 3월 기준 1,423만명), 해외주식 보유도 확대되고 있다.
  - 미국도 성인 주식 보유율이 높아(2025년 62%) 이벤트 해석 수요가 지속적이다.
  - 기존 툴은 차트/뉴스는 강하지만 "KR+US 통합 + 한국어 설명 + 근거 일관성" 조합은 아직 비어 있다.

## Technical Feasibility
- 핵심 기술 성숙도:
  - 미국 공시: SEC EDGAR API(`data.sec.gov`)는 JSON 접근이 가능하고 비인증 엔드포인트가 존재한다.
  - 한국 공시: Open DART API는 인증키 기반으로 주요 공시 조회를 제공한다.
  - 시세/뉴스: Polygon 같은 상용 피드 또는 무료/저비용 피드를 조합 가능하다.
- 난이도 추정:
  - 주말 수준: 불가(정확한 인과 추정·품질 검증 부족).
  - 6~10주: 상위 종목군 대상 MVP(감지+원인 후보+근거 링크+알림+히스토리).
  - +1개 분기: 품질 고도화(설명 정확도 추적, 잘못된 원인 피드백 루프, 비용 최적화).
- 핵심 난제:
  - 동시간대 다중 원인 분리(실적/공시/거시/수급).
  - 장전/정규장/장후, KR/US 타임존 정합.
  - "원인 추정" 문구의 법적 리스크 관리(투자자문 오인 방지).

## Competitive Landscape
- Direct:
  - Benzinga Pro: 실시간 뉴스/스캐너 강함(유료 티어 운영).
  - MarketBeat/Seeking Alpha Premium: 이벤트·리서치 제공, 원인 카드 자동화는 제한적.
- Adjacent:
  - Finviz Elite, Koyfin, TradingView: 스크리닝/차트/분석 중심.
- Open source alternatives:
  - OpenBB: 오픈소스 SDK + 상용 워크스페이스.
  - FinGPT: LLM 기반 금융 분석 프레임워크(연구/개발자 친화).
- 결론:
  - 경쟁은 매우 치열하지만 "KR+US 동시", "근거 URL 강제", "개인 보유 맥락"을 결합한 제품 포지션은 여전히 차별화 여지가 있다.

## Market Opportunity
- 초기 웨지:
  - 한국 거주 개인투자자 중 KR/US를 동시에 모니터링하고 장중 알림을 자주 확인하는 능동 투자층.
- 확장 경로:
  - B2C 구독 -> 증권사/핀테크 임베드 API(B2B2C).
- 리서치 기반 추가 기능 후보:
  - P0-1. `근거 품질 게이트`: URL 도메인 화이트리스트, dead-link 자동 제외, 동일 기사 중복 제거.
  - P0-2. `원인 카드 신뢰도 UX`: 점수뿐 아니라 "왜 이 점수인지" 가중치 설명 공개.
  - P0-3. `잘못된 원인 신고 루프`: 사용자 신고 -> 재평가 큐 -> 카드 수정 이력 노출.
  - P1-1. `What changed` 업데이트 카드: 최초 알림 이후 새 근거/점수 변화만 델타로 전달.
  - P1-2. `시드 카탈로그 강제 검색`: 임의 심볼 입력 차단 + 자동완성/최근검색/market 필터.
  - P1-3. `세션 컨텍스트 설명`: 장전/정규장/장후 별로 다른 기본 원인 템플릿 제공.
- 서비스 디테일 보강 포인트:
  - 카드 필수 필드 고정: `event_time_utc`, `exchange_tz`, `reason_type`, `confidence`, `source_url`, `published_at`, `last_updated_at`.
  - 설명 문구 표준화: 단정형 금지("~때문에 급등") 대신 확률형("~가능성이 높음").
  - 사용자 신뢰 장치: 원인 수정 이력, 데이터 지연 배지, 소스 수 부족 시 "근거 수집 중" 상태 명시.

## Risks (top 3 with mitigations)
1. 원인 정확도 미달로 신뢰 하락.
Mitigation: 신뢰도 임계치 미만 카드 비노출, 사용자 피드백을 학습/룰 튜닝에 직접 반영, 수정 이력 투명 공개.

2. 데이터 비용 상승으로 단위경제 악화.
Mitigation: MVP는 공시+핵심 뉴스 중심으로 시작하고, 유료 실시간 피드는 활성 코호트 기준 단계적 도입.

3. 강한 경쟁사 대비 차별성 약화.
Mitigation: KR 현지화(한국어 설명/국내 공시 문맥)와 포트폴리오 영향 요약을 코어로 고정해 가격경쟁이 아닌 사용시간 절감으로 승부.

## Recommendation (Go / Conditional Go / No-Go)
Conditional Go.

진행 조건:
- 파일럿(4주, 100~200명)에서 아래 KPI를 통과하면 본개발 확장:
  - 원인 카드 클릭률 >= 40%
  - 근거 URL 클릭률 >= 20%
  - "부정확함" 피드백 비율 <= 15%
  - 주간 재방문율 >= 30%
- KPI 미달 시, 기능 확장보다 먼저 `원인 품질 게이트`와 `카드 신뢰도 UX`를 보강한다.
