# Project 002: 실사용 베타 완성 (1개월)

## Goal
4주 내 프론트엔드+백엔드 통합 사용자 흐름과 운영 준비 기능을 구축해 내부/초기 사용자에게 바로 쓸 수 있는 베타를 출시한다.

## Context
- Project 001에서 P0 백엔드 이벤트 루프(관심종목/감지/원인/알림/히스토리)는 구현했지만, 인메모리 중심이며 프론트엔드 end-to-end 사용자 흐름이 없다.
- 회고에서 "백엔드 중심으로 쪼개지지 말고 실제로 테스트 가능한 화면 포함 흐름"이 핵심 개선점으로 합의됐다.
- 이번 페이즈는 PRD P1/P2 중 실사용 전환에 필요한 항목과 신규 추가된 운영 준비 요구를 우선 반영한다.
- 사용자 요청: 새 기능을 포함하되, 한 달 단위로 꼼꼼하게 당장 쓸 수 있는 완성도를 목표로 한다.

## Scope
- PostgreSQL/Redis 기반 영속화 전환, 마이그레이션, 로깅/요청 추적 등 운영 기반 구축
- 인증/인가 도입과 사용자 단위 데이터 분리
- 감지 디바운스/사용자 임계값/장전·장후 라벨링 고도화
- 원인 소스 어댑터/설명 필드/URL 검증 강화
- 인앱/이메일/푸시 알림 체계 확장
- 이벤트/피드백/포트폴리오/B2B API 확장
- Next.js 웹 UI(인증, 관심종목, 이벤트 상세, 알림/피드백/설정) 통합
- E2E/성능/보안 회귀 테스트와 CI·릴리즈 게이트 도입

## Out of Scope
- 자동매매/주문 실행
- 투자 권유/추천 모델
- 전 종목 초저지연 실시간 스트리밍(대규모 인프라)
- 네이티브 모바일 앱 출시

## Definition of Done
- All features in features.json are "done"
- All tests pass (including all previous tests)
- No regressions
